# 4장. 데이터의 개념

지금까지 우리는
- C 프로그램이 어디서 시작하는지 (`main`함수라는 입구)
- 어떤 순서로 실행되는지 (위에서 아래로 한줄씩)
- 코드가 토큰이라는 단위로 읽힌다는 것

을 배웠다.

지금까지 한 것을 요리로 치면 '조리법'을 읽는 큰 틀을 배운 것과 같다. 
조리법의 세부 내용에는 재료를 어떻게 다루는지가 적혀있다.
프로그램에게 재료란 데이터(Data)다. 

---

## 4.1 프로그램은 데이터를 다룬다

프로그램이 하는 일을 요약해보면

- 계산을 하거나
- 판단을 하거나
- 어떤 결과물을 만들어낸다

어떤 일을 하던 모든 과정에 데이터가 있다.
아무리 복잡한 코드라도 결국은 데이터를 다루기 위해 존재한다.

예를 들어 다음과 같은 상황을 생각해보자. 

```
[계산 과정]        [결과]
   1 + 2    --->    3
```

여기서 우리에게 중요한 사실은 "어떻게 CPU가 더하기를 했는가"가 아니다. 
결과적으로 "10이라는 결과를 얻었다"는 사실이 중요하다.

C 프로그램은 데이터를 불러오고, 생성하고, 저장하는 과정의 연속이다. 

## 4.2 코드에 직접 적혀있는 데이터 literal

앞 장에서 우리는 이런 코드를 본 적이 있다. 

```c
return 0;
```

여기서 `0`은 계산해서 나온 결과가 아니다. 
프로그래머가 코드 안에 **직접 적어 놓은 데이터**다. 

프로그램 코드 안에 적어 놓은 데이터를 **리터럴(Literal)**이라고 부른다. 

리터럴은

- 계산 과정을 거치지 않고
- 처음부터 "데이터 그 자체"로 존재한다
- 프로그램이 실행되는 동안 변하지 않는다 (`0`은 언제나 `0`이다)

## 4.3 데이터는 어디서 와서 어디로 가는가?

모든 데이터가 literal과 같이 한 곳에 가만히 머물러 있는 것은 아니다. 
데이터는 끊임없이 움직이며, 보통 다음과 같은 흐름을 가진다.

1. 생성
- 리터럴을 복사하거나, 파일 또는 키보드 입력에서 데이터를 읽어온다.
- 기존 데이터에 계산을 해서 새로운 데이터를 만든다.

2. 저장
- 만들어진 데이터는 사라지지 않도록 어딘가에 잠시 놓인다.
- 우리가 머릿속으로 '아까 결과가 3이었지`라고 기억하는 것과 같다.

3. 사용
- 저장해 둔 값을 꺼내서 판단을 하거나, 또 다른 계산에 사용한다.
- "그 값이 5보다 큰가?"라고 물어보는 식으로 판단을 할 수 있다.

4. 소멸
- 정해진 범위를 벗어나면 데이터는 소멸된다. 

C 언어는 데이터가 만들어지고, 저장되고, 사용되고, 소멸하는 과정을 아주 직접적이고 명확한 방식으로 표현할 수 있게 해준다.

## 4.4 데이터로 계산을 한다는 것

프로그램에서 계산이란,

> 이미 존재하는 **데이터들을 재료로** 삼아
> 새로운 데이터를 **창조**하는 과정

이다.

계산을 하면, 

- 숫자 데이터 두개를 더해서 합계라는 새로운 숫자 데이터를 만들거나
- 데이터가 같은지 비교해서 참/거짓에 해당하는 판단 결과 데이터를 만들거나
- 여러 단계를 거쳐 복잡한 계산을 할 수도 있다.

계산으로 만들어진 데이터는 

- 바로 쓰일 수도 있고,
- 나중에 다시 쓰기 위해 잠시 보관될 수도 있다.

## 4.5 데이터를 저장한다는 것

데이터는 생성되고 사용된 후에 사라진다. 하지만 프로그램에서는 **데이터를 잠시 기억해 두어야 할 때**가 정말 많다.

예를 들면,

- 힘들게 계산한 결과를 나중에 또 써야할 때
- 같은 값을 여러 곳에서 반복해서 사용할 때
- "이 값이 10보다 큰가?"하고 나중에 검사해야 할 때

데이터를 그냥 흘려보내지 않고, 필요할 때 다시 쓰려면 **어딘가에 담아 두는 방법**이 필요하다.

아직 C 언어 문법을 배우지 않았으므로, 개념적으로만 상상해보자. 마치 물건을 상자에 넣고 이름을 써 붙이는 것과 같다.

```
1. [생성] 10이라는 데이터를 만든다.
     ↓
2. [저장] 만든 데이터를 A라는 이름의 상자에 담는다.
     ↓
3. [사용] 나중에 A라는 이름을 통해 상자 안의 10을 꺼내 쓸 수 있다.
```

## 4.6 타입: 값의 의미를 결정하는 기준

앞에서 우리는
값이 만들어지고, 계산되고, 저장되고, 사용된다는 흐름을 살펴보았다.

이제 한 가지 가장 중요한 질문을 던질 차례다.

> “같은 값이라도  
> 언제나 같은 의미로 사용될까?”

C 언어에서 이 질문에 대한 답은 분명하다.

> **값의 의미는  
> 그 값을 다루는 타입에 의해 결정된다.**

---

### 값 그 자체보다 중요한 것

프로그램 안 어딘가에
어떤 값이 저장되어 있다고 생각해 보자.

예를 들어,  
어떤 위치에 `65`라는 값이 들어 있다면,

- 이것을 **문자**로 다루면 `'A'`가 될 수 있고
- **정수**로 다루면 계산에 쓰이는 숫자가 된다

중요한 점은,
값 `65` 자체가 의미를 정하는 것이 아니라,

> **그 값을 접근할 때 사용하는 타입이  
> “이 값이 무엇을 뜻하는지”를 결정한다**는 것이다.

---

### 타입은 값을 해석하는 규칙이다

C 언어에서 **타입(type)**이란,

> **저장된 값을  
> 어떤 규칙으로 해석하고 계산할지를 정하는 정보**

라고 이해하면 된다.

타입은 컴퓨터에게 다음과 같은 기준을 제공한다.

- 이 값은 정수 규칙으로 계산하는가?
- 이 값은 문자처럼 해석되는가?
- 이 값은 얼마만큼의 공간을 차지하는가?
- 어떤 연산이 허용되는가?

즉, 타입은
**값의 의미와 행동을 동시에 결정하는 기준**이다.

---

### C에서 문자는 정수로 다뤄진다

처음에는 낯설 수 있지만,
C 언어에서는 문자를 특별한 존재로 분리하지 않는다.

문자는 내부적으로
**정수 값으로 저장되고 처리**된다.

그래서 C에서는
문자 값이 정수 계산에 자연스럽게 참여할 수 있다.

이런 이유로,
문자를 나타내는 타입과
정수를 나타내는 타입은
같은 큰 묶음으로 취급된다.

---

### integral type: 정수 규칙으로 다뤄지는 타입들

C 언어에서는
다음과 같은 타입들을 한 묶음으로 다룬다.

- 문자 타입들
- 정수 타입들
- 열거형(enum)

이 타입들은 모두
**정수 규칙에 따라 계산되고 비교**된다.

이런 타입들의 묶음을
**integral type(정수 계열 타입)**이라고 부른다.

지금 단계에서는 이렇게 기억하면 충분하다.

> **integral type이란,  
> 값을 정수 규칙으로 해석하고  
> 이진수 방식으로 처리하는 타입들의 묶음이다.**

---

### 타입은 언어의 중심이다

C 언어에서 타입은
단순히 “값의 종류를 나누는 표”가 아니다.

타입은
- 값의 의미를 정하고
- 계산 규칙을 정하고
- 저장 방식과 크기를 정하고
- 프로그램의 동작 경계를 정한다

즉, 타입은
**C 언어 전체를 관통하는 사고의 기준**이다.

그래서 C를 제대로 이해하려면,
타입을 문법 암기가 아니라
**값을 해석하는 관점**으로 받아들여야 한다.

---

### 지금 단계에서 기억할 것

이 절에서 꼭 기억해야 할 핵심은 다음과 같다.

- 값의 의미는 **타입에 의해 결정된다**
- 같은 값이라도 타입에 따라 다르게 해석될 수 있다
- C에서 문자는 정수 규칙으로 처리된다
- 문자·정수·열거형은 모두 **integral type**에 속한다
- 다음 장부터는 이 타입들을 하나씩 구체적으로 배운다
