# 3장. 단어와 소스 코드의 구성

지금까지 우리는
- C 프로그램이 어디서 시작하는지
- 어떤 범위로 묶이는지
- 어떤 순서로 실행되는지

를 봤다. 

여기서는 한 단계 더 들어가서, 
**C 프로그램이 글자 단위로 어떻게 읽히는지**를 알아본다. 

---

## 3.1 C 코드의 글자들에 의미를 부여하는 단위

한글 에세이나 C 코드나 글자들과 문장 부호를 모아놓은 것이다. 

우리가 한글 문장을 읽을 때, `코딩`이라는 문자들을 보면 `코`와 `딩`으로 

따로 떼어 생각하지 않고, 띄어쓰기를 제외한 글자들을 묶어서 `코딩`이라는 하나의 단어로 인식한다.

C 코드도 정해진 규칙에 따라서 붙어 있는 문자들끼리 하나의 의미 있는 단위를 이룬다. 

프로그래밍 용어로 이를 토큰(token)이라고 부른다. 


---

## 3.2 C 코드를 토큰으로 나누는 규칙

`코드를 코딩하다?`라는 문장을 토큰으로 나눠보면 `코드를`, `코딩하다`, `?`가 된다. 

토큰으로 나눌때 쓴 규칙은 

- 띄어쓰기는 토큰을 분리한다
- 문장부호는 한글과 별개로 하나의 토큰이 된다

였다.

C 코드를 토큰으로 나눌 때도 같은 규칙이 적용된다. 다음 C 코드 

```c
int main()
{
    return 0;
}
```

을 토큰으로 나누면, 

띄어쓰기가 토큰을 분리한다는 규칙을 적용해서 `int` 토큰이 나오고, 

문장부호는 하나의 토큰이 된다는 규칙을 적용해서 `main`, `(`, `)`, `{` 토큰이 나온다.

다시 띄어쓰기가 토큰을 분리한다는 규칙을 적용해서 `return` 토큰이 나오고, 

문장부호가 하나의 토큰이 된다는 규칙을 적용해서 `0`, `;`, `}` 토큰이 나온다.


---

## 3.3 토큰의 종류

C에서 쓰이는 토큰의 종류는 크게 다음과 같이 나뉜다.

#### 키워드

우리말에서 사전에 뜻이 적혀있는 단어는 '사과', '학교', '사람'처럼 그 단어를 들으면 누구나 떠올리는 정해진 의미가 있다. C 언어 세상에서도 특별한 용도로 쓰기로 약속되어 있는 단어들이 있다.

키워드는 이미 뜻이 정해져 있기 때문에 우리가 마음대로 뜻을 바꿀 수 없다. 

지금까지 본 C 코드 중에서 `int`는 영어 단어 integer의 앞의 글자로 정수를 의미하는 키워드다. 

또한, 
```c
return 0;
```
이라는 문장이 있었는데, `return`은 여기서 실행을 끝내고 결과 값을 돌려준다는 것을 의미하는 키워드다.

#### 이름

고유 명사는 '서울', '이순신', '철수'처럼 대상에게 붙여준 고유한 이름이다. 일반 명사와 달리, 이 이름은 전통적으로 C 언어를 만든 사람들이 지어 놓았거나, 프로그래머가 새로 지을 수 있다. 

이름은 프로그래밍을 할 때 무언가를 가리킬 수 있게 해준다. 

예를 들어 프로그램의 시작점으로 쓰이는 `main`도 이름이다.

```c
int main()
{
    return 0;
}
```

### 상수

우리가 대화를 할 때 "사과 3개"라고 말하는 것처럼, C 언어로 된 문장 안에서 변하지 않는 실제 데이터 값이 있을 때, 이것을 상수라고 한다. 

예를 들어,

```c
return 0;
```
여기서 `0`은 상수이며, `return 0;`은 정상적으로 동작이 끝났다는 뜻으로 자주 사용된다.

### 문장 부호

우리말에서 단어만 나열하면 문장을 이해하기 어렵듯이, C 코드에서도 어디가 문장의 끝이고, 어디가 한 묶음인지 표시해주는 문장 부호들이 있다. 

예를 들어, 

```c
return 0;
```
여기서 `;`가 문장 부호이며, 문장의 끝을 의미한다.

```c
int main()
{
    return 0;
}
```
여기서 `( )`는 `main`을 실행할 때 필요한 입력에 대한 내용을 묶어서 표현하는데 쓰이는 문장 부호고, 

`{ }`는 함께 실행되는 문장들을 묶는데 쓰이는 문장 부호다.

## 3.4 공백과 줄바꿈은 어떻게 취급될까

C 코드를 읽거나 쓰다 보면, 

- 띄어쓰기(space)
- 줄바꿈(newline)
- 탭(tab)

같은 것들이 많이 보인다.

하지만 대부분의 경우, 이들은 토큰을 구분하기 위한 역할만 한다. 

즉, 다음 두 코드는 의미가 같다.

```c
return 0;
```
```c
return      0
;
```
두번째 코드는 사람이 보기에 이상할 수 있지만, 컴퓨터는 같은 토큰으로 해석한다.

우리는 컴퓨터가 아니라 사람을 위해 공백과 줄바꿈을 적절하게 사용한다.

## 3.5 잠깐: `#include`로 시작하는 줄은 어떻게 해석해야 할까?

지금까지 우리는 C 프로그램이 **토큰(token)**이라는 단위로 나뉘어 읽힌다는 것을 배웠다.

그렇다면 이런 줄은 어떻게 읽힐까?

```c
#include <stdio.h>
```

`#include`처럼 `#`으로 시작하는 줄은 일반적인 C 코드와 같은 방식으로 바로 읽히지 않는다.
대신, 코드를 읽기 전에 먼저 처리되는 특별한 줄이다.

`#`으로 시작하는 줄은
- 실행되는 동작을 직접 적어 놓은 것이 아니라
- 코드를 어떻게 준비할 것인가를 알려주는 역할을 한다.
  
그래서 지금 배우고 있는 토큰 규칙의 범위 바깥에 있다.

코드를 준비하는 단계를 프로그래밍 용어로 **전처리(preprocessing)**이라고 부른다.

## 3.6 지금까지 만난 토큰들

지금까지 우리는 다음과 같은 토큰들을 봤다.

- 키워드: C가 미리 정해 둔 특별한 단어 (`return`, `int` 등)
- 식별자: 프로그래머가 붙이는 이름 (`main` 등)
- 상수: 코드에 직접 적힌 값 (`0` 등)
- 문장부호: 문장의 끝이나 구조를 표시하는 기호 (`;`, `{ }`, `( )` 등)

앞으로 새로운 종류의 토큰이 나오면 그때마다 하나씩 추가로 배우면 된다.
