# 12. C 프로그램에서 글자의 의미: 입력과 출력

우리는 지금까지 함수를 만들고, 함수 안에 정수 타입, 주소 타입, 배열 타입의 변수를 만들어서 값을 저장하고, 산술 연산자 `+`, 주소 연산자 `&`, 배열 인덱스 연산자 `[]`로 계산 하거나 함수 실행 연산자`()`로 함수를 실행하고, `if`와 `for`로 복잡한 작업도 해봤다.

지금까지 우리가 익힌 C 문법으로 만든 프로그램은 외부에는 실행 결과로 에러 코드를 돌려주기만 할 뿐, 모든 동작이 내부에서만 이뤄졌다. 

그런데 실제 프로그램에서는 프로그램이 실행되면 화면에 그림을 그리고, 키보드와 마우스로 입력을 받아서 동작한다. 

유저와 상호작용하는 가장 간단한 방법인 글자 입출력을 다뤄보자. 

## 12.0 이 장의 목표

이 장을 끝내면 다음을 이해하고 구현할 수 있다. 

1. **글자 관련 라이브러리 함수:** 글자를 화면에 출력하고, 키보드 입력을 받을 수 있다.
2. **영어 글자를 저장할 수 있는 타입:** 영어 대소문자와 특수문자를 저장할 수 있는 크기의 타입 `char`를 써본다.
3. **문장을 저장하기:** `char` 배열에 영어 문장을 저장해본다.
4. **문장 처리하기:** 배열 작업을 통해 문장을 처리해본다.
5. **문장 관련 라이브러리 함수:** 문장을 포맷팅하고, 화면에 출력해본다.

## 12.1 한 글자씩 주고받기: `getchar`와 `putchar`

C 언어에서 제공해주는 함수인 `getchar`를 실행하면, 키보드에서 키를 입력받을 수 있다. 또 다른 C 언어에서 제공해주는 함수인 `putchar`를 실행하면, 컴퓨터 화면 _terminal_ 에 글씨를 쓸 수 있다. `getchar`와 `putchar` 함수의 선언은 `<stdio.h>`에 들어있고 함수 본문은 C 언어 자체에 내장되어 있어서, 소스 코드에 `#include <stdio.h>`만 하면 두 함수를 쓸 수 있다. 

- **`int getchar(void)`:** 키보드에서 어떤 키를 하나 누르면 거기에 해당하는 숫자를 반환한다. 실행 중 문제가 있었으면 음수가 반환된다.
- **`int putchar(int c)`:** `c`에 해당하는 글자를 화면에 출력한다. 실행 중 문제가 없었으면 `c`가 그대로 반환되고, 문제가 있었으면 음수가 반환된다.

> [!NOTE]
> **입력 끝을 나타내기: _end of file, EOF_**
>
> `getchar` 함수는 더 이상 읽을 입력이 없거나 읽기 과정에 문제가 생기면, `<stdio.h>`에 정의된 `EOF` 라는 특별한 기호를 반환한다. `EOF`는 보통 **-1**의 값을 가진다.
>
> **키보드에서 `EOF`를 쓰는 방법은?**
>
> 키보드에는 `EOF`라는 키가 없다.
> 윈도우에서는 Ctrl + 'Z' 키를, 맥OS나 리눅스에서는 Ctrl + 'D' 키를 입력한 다음에
> 엔터 키를 누르면 프로그램에 `EOF` 신호가 전달된다. 

### 12.1.1 키보드로 키 입력받기: `getchar`

키보드로 입력받은 키를 변수에 저장하는 코드를 써보자. `getchar`를 실행하기 위해서는 `#include <stdio.h>`를 먼저 해야한다. `#include`는 전통적으로 소스 파일에서 제일 처음에 적는다. 

```c
#include <stdio.h>

/* getchar 함수 실행해보기 */
int main(void)
{
        int key = getchar();
        return 0;
}
```

중단점을 `return 0;`에 설정한 다음에 디버깅을 시작해보면, 프로그램 실행이 `int key = getchar();`에서 멈춰있는 것을 볼 수 있다. 키보드에서 키를 하나 누르고 엔터를 누르면 `return 0;` 앞까지 진행이 되고, 변수 `key`에 입력한 키에 해당하는 숫자가 저장된다.

> [!NOTE]
> **엔터 _enter_ 키의 진실: 줄 단위 버퍼링 _line buffering_**
>
> `getchar`를 했을 때, 실제 터미널 창에서는 'a' 라는 키를 눌러도 프로그램이 곧바로 진행되지 않고
> 엔터 키를 눌러야만 다음 코드로 넘어간다.
> 그 이유는 터미널에서 사용자가 오타를 수정할 기회를 주기 위해 엔터를 칠 때까지 입력을 임시로 모아두는
> 버퍼 _buffer_ 를 사용하기 때문이다.
> 
> 그래서 'a' 키를 누르고 엔터 키를 누르면, 'a'와 줄바꿈을 의미하는 문자가 세트로 입력된다.
>
> ```c
> #include <stdio.h>
>
> /* 엔터 키 입력 확인하기 */
> int main(void)
> {
>         int key1 = getchar();
>         int key2 = getchar();
>         return 0;
> }
> ```
>
> 위 프로그램을 실행한 후에, 'a' 라는 키를 누르고 엔터 키를 누르면,
> key1에는 'a'에 해당하는 숫자 **97**이, key2에는 엔터에 해당하는 숫자 **10**이 저장된다.

### 12.1.2 화면에 글자 출력하기: `putchar`

화면에 글자를 출력하는 코드를 써보자. `putchar`를 하기 위해서는 입력값을 정해줘야 하는데, `97`이라고 적어보자. 

```c
#include <stdio.h>

int main(void)
{
        putchar(97);
        return 0;
}
```

어떤 숫자가 어떤 글자를 의미하는지는 C 표준에 정해져 있지는 않지만, `97`은 대부분 컴퓨터에서 영어 소문자 'a'를 의미한다. 

> [!NOTE]
> **글자에 붙여놓은 정수: _American Standard Code for Information Interchange, ASCII_**
>
> 알파벳과 특수문자를 어떤 정수로 나타낼지를 컴퓨터마다 통일하기 위해 미국에서 정한 표준이다. 예를 들어, 소문자 'a'는 97, 대문자 'A'는 65, 숫자 '0'은 48로 나타내기로 정해져 있다. 전 세계의 거의 모든 컴퓨터가 이 표준을 따른다.
>
> 그렇다면 한글 글자는 어떨까?
> 유니코드 _unicode_ 에서 전 세계 모든 언어의 글자를 숫자로 나타내는 표준을 정하고 있다.
> 인터넷에서 표준인 _UTF-8_ 이 유니코드를 표현하는 방식 중 하나다. 

## 12.2 글자를 코드에 적는 방법: 작은따옴표 `''`

`putchar`로 화면에 어떤 글자를 출력할지를 정하고, `getchar`로 입력 받은 글자가 무엇인지 확인하기 위해서
프로그램이 실행될 컴퓨터마다 각 글자가 어떤 정수 값으로 표현되는지를 항상 미리 알아두어야 할까?

C 언어에서는 해당 숫자를 외우지 않고도 글자를 표현할 수 있는 방법을 만들어 놓았다. 글자를 작은 따옴표 `''`로 감싸는 방법이다. 컴파일러는 코드에서 `'a'`를 발견하면, 알아서 그 컴퓨터의 표준에 해당하는 정수인 `97`로 바꿔서 계산해준다.

작은 따옴표 `''`를 씀으로써 코드를 읽는 사람에게 의도를 더 명확하게 전달할 수 있고, 어떤 컴퓨터에서든 의도한 글자가 안전하게 출력되도록 코드를 짤 수 있다.

```c
#include <stdio.h>

/* 화면에 abc를 출력한다 */
int main(void)
{
        putchar('a');
        putchar('b');
        putchar('c');
        return 0;
}
```

`getchar`를 할 때도, 작은 따옴표 `''`로 글자를 표현하면, 입력받은 키가 특정 글자인지 명확하게 확인할 수 있다. 

```c
#include <stdio.h>

/* 입력받은 키를 그대로 출력한다. 'q'를 누르면 종료된다 */
int main(void)
{
        for (;;) {
                int key = getchar();
                putchar(key);

                if (key == 'q') {
                        break;
                }
        }
        return 0;
}
```

위의 프로그램은 입력받은 내용을 그대로 화면에 출력하는 메아리 _echo_ 프로그램이다. 
프로그램을 실행하고 `abq`를 입력한 뒤 엔터 키를 눌러보자. 
앞에서 봤듯이 터미널은 엔터 키를 누를 때까지 모든 입력을 버퍼에 모아두고 있기 때문에, 
엔터키를 누르면 `a`, `b`, `q`, `엔터`가 한번에 프로그램에 전달된다.

`getchar` 함수는 전달받은 글자들에서 앞에서부터 차례대로 하나씩 읽어서 반환한다.

1. **첫번째 반복:** `'a'`를 읽어와서 출력한다. `q`가 아니므로 반복을 계속 진행한다.
2. **두번째 반복:** `'b'`를 읽어와서 출력한다. `q`가 아니므로 반복을 계속 진행한다.
3. **세번째 반복:** `'q'`를 읽어와서 출력한다. `q`를 발견했으므로 반복을 종료한다.

결과적으로 화면에 `abq`가 출력되고 프로그램이 종료된다. 
프로그램이 종료되며 버퍼에 남아있던 `엔터` 글자는 읽히지 않고 없어진다.

## 12.3 특수기호를 쓰는 방법: `'\n'`, `'\t'`, `'\0'`, `'\\'`, `'\''`

`엔터` 글자를 소스 코드에 적으려고 할때, 알파벳처럼 작은 따옴표 `''`로 감싸는 방식으로는 표현할 수 없다.
키보드로 직접 쓰기 어렵거나 특별한 의미를 가진 글자를 표현하기 위해 C 언어는 역슬래시 `\`를 써서 원래 뜻과 다른 의미를 부여하는 _escape sequence_ 표기법을 제공한다.

| 표기 | 뜻 | 많이 쓰는 곳 |
| :---: | :--- | :--- |
| `'\n'` | 줄바꿈 _newline_ | 다음 줄로 바꾸기 |
| `'\t'` | 탭 _tab_ | 간격 맞추기 |
| `'\0'` | 정수값이 **0** | 없는 글자 _null_ 을 의미 |
| `'\\'` | 역슬래시 | 역슬래시 `\` 글자를 쓸 때 |
| `'\''` | 작은따옴표 | 작은따옴표 `'` 글자를 쓸 때 |

줄바꿈 _newline_ 과 탭 _tab_ 의 경우 역슬래시 `\` 뒤에 영어 단어의 앞 글자인 `n`과 `t`를 붙여서 나타낸다. 

`'\0'`은 정수 값 **0**을 가지는 특별한 글자다. 주로 여러 글자를 이어서 쓸 때, 더이상 글자가 없음을 알려주는 마침표 역할을 한다.

또한, 역슬래시 `\`와 작은따옴표 `'`는 글자 문법에서 이미 특수한 목적으로 사용되고 있는 기호들이다. 그래서 이 글자들 자체를 표현할 때는 혼동을 막기 위해 앞에 역슬래시를 붙여서 `'\\'`와 `'\''`라고 적는다.

다음 프로그램을 실행하면 화면에 무엇이 출력될까? 코드를 한 줄씩 읽으며 결과를 예상해보자.

```c
#include <stdio.h>

/* 특수 기호를 화면에 출력하는 프로그램 */
int main(void)
{
        putchar('\'');
        putchar('a');
        putchar('\'');
        putchar('\n');

        putchar('\'');
        putchar('\\');
        putchar('t');
        putchar('\'');

        return 0;
}
```

프로그램을 실행하면, 화면에 다음과 같이 출력된다. 

```text
'a'
'\t'
```

역슬래시를 활용한 덕분에 문법 기호인 작은따옴표와 역슬래시가 안전하게 화면에 그려진 것을 볼 수 있다.

## 12.4 글자를 저장하기에 알맞은 크기의 타입: `char`

지금까지 우리가 주로 쓰던 타입인 `int`는 컴퓨터에 따라 16비트 또는 32비트의 크기를 가진다. 16비트로는 대략 6만 가지의 값을, 32비트로는 대략 40억 가지의 값을 표현할 수 있다. 

우리가 이번 장에서 다루고 있는 글자를 저장하기 위해서는 몇 비트가 필요할까? C 언어 소스 코드를 적거나 일반적인 영어 문서를 작성하는데 필요한 글자는 영어 대소문자, 숫자, 기호를 모두 합쳐도 대략 100개 남짓이다. 따라서 8비트만 있어도 소스 코드나 영어 문장을 적기 위한 모든 글자를 표현하기에 충분하다. 

C 언어는 메모리를 효율적으로 사용하기 위해, 딱 8비트, 다른 말로는 1바이트 크기만 차지하는 가장 작은 정수 계열 타입인 `char` 타입을 지원한다. `char`는 영어로 글자를 의미하는 단어 _character_ 의 앞의 글자들을 따서 정해졌다. 

> [!CAUTION]
> **`char`에 쓸 수 있는 안전한 범위: 0부터 127까지**
>
> 8비트로 표현할 수 있는 값은 256가지이다.
> 그런데 컴파일러에 따라 `char`가 표현하는 범위가 -128부터 127까지이기도 하고, 0부터 255까지이기도 하다.
> 따라서 어떤 컴퓨터에서도 안전하게 쓸 수 있는 범위는 0부터 127까지의 숫자로, 표준 _ASCII_ 의 범위와도 같다.
> 달리 말하면, 작은따옴표 `''`로 감싼 알파벳, 숫자, 기호 등의 기본 글자들은 `char` 타입 변수에 언제나 안전하게 저장할 수 있다.

> [!NOTE]
> **바이트: 주소가 부여되는 메모리의 최소 단위**
>
> 컴퓨터 메모리 내부에는 0과 1을 저장할 수 있는 수많은 비트 _bit_ 라는 작은 스위치들이 존재하지만,
> CPU가 모든 비트 하나하나를 개별적인 주소로 찾아갈 수는 없다.
> 대신, 스위치 8개를 묶어 하나의 방인 바이트 _byte_ 를 만들고, 방 단위로 고유한 주소 _address_ 를 부여한다.
> `char` 타입은 방 한칸에 해당하는 1 바이트의 크기를 갖기 때문에
> 낭비되는 공간 없이 글자 하나를 딱 맞게 집어넣을 수 있는 가장 효율적인 데이터 타입이다. 

새로 알게된 `char 타입을 써서 화면에 글자를 출력하는 프로그램의 소스 코드를 써보자.  

```c
#include <stdio.h>

/* 화면에 abc를 출력한다. */
int main(void)
{
        char c = 'a';
        putchar(c);

        c = 'b';
        putchar(c);

        c = 'c';
        putchar(c);

        return 0;
}
```

> [!CAUTION]
> **주의: `getchar`의 결과값은 `char` 상자에 저장하면 안된다**
>
> `getchar`를 했을때 입력의 끝에 도달하거나 문제가 발생하면 반환되는 `EOF`는 음수이기 때문에,
> `char` 타입에 안전하게 저장할 수 있는 범위를 벗어난다.
> `getchar`의 결과는 `int`에 저장해야 한다. 

## 12.5 문장을 표현하는 방법: 글자 여러개 묶기

여러 글자로 이루어진 문장은 메모리에 어떻게 저장할까? 우리는 8장에서 배열을 사용해서 같은 타입의 데이터를 연속적으로 나열할 수 있다는 것을 알게 되었다. 문장 역시 `char` 크기의 방 여러 개를 붙여놓고, 글자를 차례대로 하나씩 저장하면 된다.

배열을 쓸 때 항상 고려할 점은 어디까지가 실제 데이터인지를 컴퓨터에게 알려줘야 한다는 것이다. 일반적인 숫자 배열에서는 저장된 데이터의 총 개수를 별도로 변수에 기록해두는 방식을 주로 사용했다.

하지만 글자들을 다룰 때는 상황이 조금 다르다. 프로그램에서 문장을 처리할 때는 긴 문장을 짧은 단어들로 분해해서 메모리에 저장하는 경우가 많다. `cd`와 같이 글자 수가 2개인 단어를 저장하려고 하는데, 데이터 2바이트에 추가로 글자 수를 기억하기 위한 `int` 변수 4바이트를 쓰면 어떨까? 실제 데이터보다 메타 데이터를 저장하는 데에 메모리를 더 많이 소모하는 낭비가 발생한다. 

그래서 C 언어에서는 문장을 `char` 배열에 저장할 때 글자 개수를 따로 저장하는 대신, 문장의 끝에 없는 _null_ 글자 `'\0'`를 적어서 표시하는 방법을 택했다.

> [!CAUTION]
> **보안 관점: 마침표 `'\0'`을 잊어버리면 벌어지는 일**
>
> 메모리를 효율적으로 쓰기 위해 도입된 방식이지만,
> 프로그래머가 실수로 배열의 끝에 `'\0'`을 적지 않으면 컴퓨터는 문장이 끝났다는 사실을 알지 못한다.
> 결국 컴퓨터는 `'\0'`을 발견할 때까지 배열의 범위를 넘어 이웃한 메모리 방을 계속 읽어들이게 되고,
> 이는 프로그램이 엉뚱한 값을 출력하거나 프로그램이 멈춰버리는 치명적인 오류로 이어진다.

`char` 배열을 써서 문장을 만들고, 화면에 출력하는 프로그램을 만들어보자.

```c
#include <stdio.h>

/* 화면에 Hello를 출력하는 프로그램 */
int main(void)
{
        char str[6] = {'H', 'e', 'l', 'l', 'o', '\0'};

        for (int i = 0; str[i] != '\0'; i++) {
                putchar(str[i]);
        }

        return 0;
}
```

프로그램이 실행되면 메모리에 만들어진 `str` 배열 안에는 다음과 같이 데이터가 저장된다.

| 메모리 위치 | `str[0]` | `str[1]` | `str[2]` | `str[3]` | `str[4]` | `str[5]` |
| :---: | :---: | :---: | :---: | :---: | :---: | :---: |
| **문자** | `'H'` | `'e'` | `'l'` | `'l'` | `'o'` | `'\0'` |

반복문은 0번째 칸의 `'H'`부터 시작해 한 칸씩 이동하며 글자를 화면에 그리다가, 5번째 칸에서 `'\0'`을 만나면 즉시 출력을 멈추고 종료된다.

## 12.6 문장을 코드에 적는 방법: 큰따옴표 `""`

프로그래밍을 할 때 문장을 다룰 일이 많다.
그래서 C 언어는 소스 코드에 문장을 쉽게 적을 수 있는 문법을 제공한다. 
여러 글자들을 큰따옴표 `""`로 감싸는 방법이다.
큰따옴표 안에 글자들을 적어 놓으면 제일 끝에 마침표 역할을 하는 `'\0'` 글자는 컴파일러가 알아서 추가해준다.

> [!NOTE]
> **소스 코드에 적은 데이터: 리터럴 _literal_**
>
> 프로그래밍에서 숫자 `5`나 글자 `'A'`, 그리고 지금 본 `"Hello"`처럼 소스 코드에 고정된 값으로 직접 적은
> 데이터를 리터럴 _literal_ 이라고 부른다. 변수 상자 _variable_ 는 상황에 따라 저장된 값이 변할 수 있지만,
> 리터럴은 코드에 적힌 형태 그대로 고정되어 있다.
>
> - **정수 리터럴:** `10`, `-255`
> - **문자 리터럴:** `'a'`, `'\n'`
> - **문자열 리터럴:** `"Hello"`

### 12.6.1 큰따옴표 `""`의 용도: `char` 배열의 초기값

C 언어 문법에서, 큰따옴표 `""`를 쓰는 용도 중 하나는 `char` 배열의 초기값을 정해주는 것이다. 큰따옴표 `""`를 쓰면 `char` 배열을 초기화할 때 `{'H', 'e', 'l', 'l', 'o', '\0'}`처럼 쓰던 것을 `"Hello"`처럼 간단하게 쓸 수 있다. 

```c
/* 큰따옴표로 Hello 문장 만들기 */
char str[6] = "Hello";
```

여기서 주의할 점은, `"Hello"`는 5글자이지만, 실제로는 끝에 `'\0'`이 붙어서 저장되기 때문에 배열의 크기가 **6**이 되어야 한다는 것이다. 

> [!CAUTION]
> **보안 관점: 문장을 담을 배열의 크기가 부족하면 벌어지는 일**
>
> 문자열 리터럴로 `char` 배열을 초기화할 때 초보자들이 가장 많이 하는 실수가 있다.
> 바로 `char str[5] = "Hello";`처럼 글자 수만큼만 메모리 공간을 할당하는 것이다.
>
> 그러면 컴퓨터는 `'H'`, `'e'`, `'l'`, `'l`', `'o'` 5글자를 `str` 배열에 꽉 채워 넣는다. 문제는 마지막에 `'\0'`을 저장할 공간이 없다는 것이다. C 언어는 마지막에 `'\0'`을 저장할 공간이 없는 경우 경고 없이 그냥 `'\0'`을 버린다.
>
> 결국 이 배열은 마침표를 잃어버리게 되고,
> 나중에 화면에 출력하거나 문장을 분석할 때
> `'\0'`을 찾지 못해 배열 밖의 메모리를 침범하는 메모리 초과 읽기 버그를 일으키게 된다.

**배열 크기를 컴파일러에게 맡기기**

8장에서 배열을 만들 때 초기값을 적어주면, 컴파일러가 알아서 데이터 개수를 세어 배열 크기를 알아서 정해준다고 했었다. 같은 규칙이 큰따옴표 `""`로 `char` 배열을 초기화할 때도 적용된다. 

```c
/* 배열 크기는 컴파일러가 알아서 적절하게 정해준다 */
char str[] = "Hello";
```

위 코드와 대괄호 `[]` 안을 비워두고 쓰면, 프로그래머가 직접 글자 개수를 세다가 마침표 자리를 빼먹는 실수를 원천적으로 차단할 수 있다. 

### 12.6.2 주의: 배열을 만들 때가 아니면 큰따옴표 `""`로 문장을 저장할 수 없다

큰따옴표 `""`가 너무 편리하다 보니, 초보자들은 문장을 다룰 때 일반적인 `int` 상자처럼 
언제든지 `=` 기호로 새로운 문장을 저장할 수 있다고 착각하기 쉽다. 다음 코드를 보자.

```c
/* 초보자가 많이 하는 실수 */
char str[10];
str = "Hello";  /* 컴파일 에러 발생! */
```

이 코드는 컴파일 에러를 발생시킨다. 일반적으로, C 언어의 수식에서 배열의 이름 `str`은 만들어진 메모리 방들의 시작 주소값을 의미하고, 문자열 리터럴 `"Hello"`도 배열이기 때문에 메모리 방들의 시작 주소값을 의미한다. 주소값 자체는 바꿀 수 없으므로, `str`의 시작 주소값에 `"Hello"`의 시작 주소값을 대입할 수 없다.

앞에서 썼던 `char str[] = "Hello";`는 배열을 만들며 초기값을 정하는 _initialization_ 상황에서 비어있는 방에 글자들을 차례대로 복사해서 저장해주는 특별히 허용된 문법이었다.

**그렇다면 이미 만들어진 배열의 문장을 바꾸려면 어떻게 해야할까?**

만약 배열을 만든 이후에 문장을 바꾸고 싶다면, 큰따옴표를 쓸 수 없고 처음 배웠던 방식처럼 배열의 각 칸에 직접 접근해서 글자를 하나씩 바꿔주어야 한다. 

```c
char str[10] = "Apple";

/* 다음과 같이 통째로 바꾸는 것은 불가능하다 */
/* str = "Hi"; */

/* 글자를 하나씩 직접 바꿔야 한다 */
str[0] = 'H';
str[1] = 'i';
str[2] = '\0'; /* 마침표를 잊지 말자 */
```

### 12.6.3 읽기 전용 문장 가리키기: `const`와 `const char *`

문자열 리터럴 `"Hello"`는 결국 배열이기 때문에 시작 주소값을 가진다고 했다. 
그렇다면 배열을 새로 만들어서 복사하는 대신, 포인터 변수를 만들어서 `"Hello"`의 시작 주소를 바로 가리키게 할 수는 없을까?


가능하다. 하지만 이 방식에는 치명적인 함정이 숨어있다.

운영체제는 프로그램이 실행될 때, `"Hello"`와 같은 문자열 리터럴 데이터들을 메모리의 특별한 구역에 모아둔다. 
리터럴은 프로그램이 실행되는 동안 내용이 바뀌어서는 안 되기 때문에 읽기 전용 _read-only_ 으로 강력하게 보호받는다.

만약 포인터로 읽기 전용 구역을 가리킨 다음, 글자를 바꾸려고 시도하면 어떻게 될까?

```c
/* 대단히 위험한 코드 */
char *ptr = "Hello";        /* ptr은 읽기 전용 구역에 있는 'H'의 주소를 가진다. */
ptr[0] = 'M';               /* 읽기 전용 메모리를 수정하려고 시도하면, 프로그램이 강제 종료될 수 있다 */
```

배열 `char str[] = "Hello";`는 내 소유의 메모리 방을 만들고 거기에 글자들을 복사해 온 것이기 때문에
`str[0] = 'M'`처럼 마음대로 수정할 수 있었다.
하지만 포인터 `char *ptr = "Hello";`는 원본이 있는 읽기 전용 전시관을 손가락으로 가리키고 있는 상태다. 
전시관의 작품을 마음대로 훼손하려고 하면 운영체제가 즉시 개입하여 프로그램을 강제로 종료시켜 버린다 _segmentation fault_. 

**안전하게 가리키는 방법:** `const char *`

C 언어에는 상자를 읽기 전용으로 만들어주는 `const`라는 키워드가 있다. 
`const`는 변하지 않는다는 _constant_ 의 약자로, 
변수의 값이나 포인터가 가리키는 값을 절대 수정하지 않겠다고 컴파일러에게 약속하는 문법이다. 

문자열 리터럴의 주소를 포인터에 저장할 때는 항상 `const`를 붙여서 `const char *` 타입으로 선언하는 것이 
안전한 프로그래밍 습관이다. 

```c
/* 안전한 방식 */
const char *ptr = "Hello";

/* 가리키는 곳에 저장된 값을 변경하려고 하면 컴파일러가 에러를 띄워서 실수를 미리 막아준다 */
/* ptr[0] = 'M'; */

/* 가리키는 대상을 다른 문장으로 바꾸는 것은 가능하다 */
ptr = "Hi";
```

`const char *ptr`로 선언해두면, 
실수로 `ptr[0] = 'M'` 처럼 글자를 수정하려고 할 때 컴파일러가 빨간 줄을 그으며 막아준다.
전시관에 있는 `"Hello"` 작품 대신 다른 작품인 `"Hi"`를 가리키도록 손가락의 방향을 바꾸는 `ptr = "hi';`는 허용된다.

> [!NOTE]
> **`char []` vs `const char *`**
>
> - **`char str[] = "Hello";`:** 내 방을 만들고 문장을 복사해온다. 글자 수정 가능하다.
> - **`const char *ptr = "Hello";`:** 읽기 전용 원본을 가리킨다. 글자 수정은 불가능하고 가리키는 대상 변경은 가능하다.

## 12.7 문자열 입출력 함수 만들어보기: `print_string`과 `read_string`

C 언어는 문장을 쉽게 다룰 수 있는 다양한 표준 라이브러리 함수들을 제공하지만, 
그 함수들을 사용하기 전에 우리가 직접 입출력 함수를 만들어보자. 
지금까지 배운 `for` 반복문과 `putchar`, `getchar`, 그리고 마침표 `'\0'`의 원리만 알면 충분히 만들 수 있다.
직접 함수를 만들어보면 문장이 메모리에서 어떻게 처리되는지 확실하게 이해할 수 있다.

### 12.7.1 문장 출력 함수 만들기: `print_string`

먼저 문장을 화면에 출력하는 `print_string` 함수를 만들어보자. 
이 함수는 배열의 시작 주소를 넘겨받은 뒤, 
`'\0'`을 만날 때까지 `for` 반복문으로 배열의 모든 상자들에 대해 반복하며 `putchar`로 글자를 하나씩 출력한다.

입력값을 받을 때는 `const` 키워드를 써보자. 
출력 함수는 문장을 읽기만 할 뿐 수정할 일은 없으므로, 입력값을 `const char *`로 받는 것이 안전하다. 

```c
#include <stdio.h>

/* 문장을 화면에 출력하는 함수 */
void print_string(const char *str)
{
        /* 시작 주소부터 한 칸씩 이동하며 '\0'이 아닐 때까지 반복해서 화면에 추력한다 */
        for (int i = 0; str[i] != '\0'; i++) {
                putchar(str[i]);
        }
}

/* 화면에 여러 문장을 출력하는 프로그램 */
int main(void)
{
        char my_str[] = "Hello C!";

        print_string(my_str);
        print_string("\n");        /* 줄바꿈 출력 */
        print_string("Good!");     /* 문자열 리터럴을 직접 넘겨도 잘 동작한다 */
        print_string("\n");        /* 줄바꿈 출력 */

        /* 실행 후 화면에 출력되는 내용:
         * Hello C!
         * Good!
         */
```

입력값을 `const char *str`로 선언한 덕분에, `print_string(my_str);`처럼 내가 만든 배열을 넘겨줄 수도 있고,
`print_string("Good!");`처럼 읽기 전용인 문자열 리터럴을 직접 넘겨도 에러 없이 안전하게 동작한다. 

### 12.7.2 문장 입력 함수 만들기: `read_string`

이번에는 키보드에서 문장을 입력받아 배열에 저장하는 `read_string` 함수를 만들어보자. 
`getchar`를 사용해 엔터 키 `'\n'`를 누를 때까지 글자를 하나씩 입력받아 배열에 차례대로 저장하면 된다.

여기서 주의할 점이 두 가지 있다.

- **메모리 초과 방지:** 유저가 배열의 크기보다 더 많은 글자를 입력하지 못하도록 최대 길이를 확인해야 한다.
- **마침표 추가:** 엔터를 쳐서 입력이 끝난 다음에는, 끝에 마침표 `'\0'`를 직접 저장해야 한다.

```c
#include <stdio.h>

/* 키보드 입력을 최대 max_length개 받아서 str에 저장하는 함수. */
void read_string(char str[], int max_length)
{
        int i;

        /* 끝에 마침표 '\0'가 들어갈 한칸을 남겨두기 위해 max_length - 1까지 반복 */
        for (i = 0; i < max_length - 1; i++) {
                int key = getchar();

                /* 엔터 키를 누르거나 입력이 끝나면 반복 중단 */
                if (key == '\n' || key == EOF) {
                        break;
                }

                str[i] = key; /* 배열에 글자 저장 */
        }

        /* 반복이 끝난 후, 직접 마침표를 저장해줘야 완벽한 문장이 된다 */
        str[i] = '\0';
}

/* 키보드 입력을 받고, 입력받은 내용을 그대로 다시 출력해서 확인하는 프로그램 */
int main(void)
{
        /* 10칸 짜리 상자를 준비한다. 최대 9글자 + 마침표 1글자 */
        char buffer[10];

        print_string("문장을 입력하세요: ");

        /* buffer와 그 크기인 10을 함께 넘겨준다 */
        read_string(buffer, 10);

        print_string("입력된 문장: ");
        print_string(buffer);

        print_string("\n");

        return 0;
}
```

> [!NOTE]
> **보안 관점: 입력 버퍼에 남은 찌꺼기**
>
> `read_string` 함수는 유저가 `"programming"` 처럼 긴 단어를 입력하면 9글자 `"programmi"` 까지만 저장하고 반복을 종료한다.
> 
> - 남은 글자인 `"ng"`와 유저가 누른 엔터키 `'\n'`가 터미널의 입력 버퍼에 그대로 남아있게 된다.
> - 만약 입력을 또 받기 위해서 `read_string`이 실행된다면, 버퍼에 남아있던 "ng\n"를 다음 변수에 저장하고,
>   새로 키보드 입력을 받지 않게 된다.
>
> 따라서 안전하게 프로그래밍을 하려면 엔터키가 나올 때까지 남은 글자들을 읽는 작업이 추가되어야 한다.

> [!NOTE]
> **보안 관점: 비정상적인 입력값 방어**
>
> 만약 함수를 실행할 때 실수로 `max_length`에 0이나 음수를 넣어서 실행하면 어떻게 될까?
> 반복은 실행되지 않고 멈추지만, `str[i] = '\0';` 코드가 무조건 실행되기 때문에,
> 허락되지 않은 배열의 0번째 칸에 접근하는 메모리 침범 버그가 발생한다.
>
> 만약 `str`에 `NULL`을 전달해서 실행하면 어떻게 될까?
> 잘못된 주소인 `str[i]`에 접근하려다가 프로그램이 강제 종료 _segmentation fault_ 된다.
>
> 잘못된 입력값으로 인한 오동작을 막기 위해서는, 함수를 시작할 때 입력값이 비정상적인 값이면 함수를 종료하는 방어벽이 필요하다.
