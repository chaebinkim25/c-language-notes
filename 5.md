# 5장. 정수 계열 타입: int와 기본 연산

앞장에서 우리는 데이터가 생성되고, 계산되고, 저장되고, 사용되고, 사라지는 흐름을 다뤘다. 

여기서는 그 흐름을 **가장 자주 쓰이는 `int`데이터 타입**을 써서 실제 코드로 구현해 본다.


> 목표: 6장의 `if`와 7장의 `for`를 할 수 있을 만큼의 **최소한의 `int` + `int`에 적용되는 최소한의 연산자**만 익힌다.  
> 그래서 이 장에서는 **`char`, `short`, `long`, `signed/unsigned`, `enum`은 일부러 미룬다.**
> 복잡한 연산자 (`&`, `|`, `~`, `++`, `--`, `? :`, `,`) 또한 나중으로 미룬다.

---

## 5.1 왜 하필 `int`부터 시작하나?

우리가 프로그램을 만들 때 가장 빈번하게 수행하는 작업들은 보통 다음과 같다.

- 세기 (Counting): "하나, 둘, 셋..." 숫자를 센다.
- 반복 (Iteration): "이 작업을 10번만 해라."
- 크기/개수 (Size/Quantity): 데이터가 몇 개인지, 용량이 얼마인지 다룬다.
- 분기 (Condition): 값을 비교해서 이쪽 길로 갈지 저쪽 길로 갈지 정한다.
- 종료 (Termination): "0이 되면 멈춰라" 같이 끝내는 조건을 만든다.

이 작업들의 공통점은 전부 '딱 떨어지는 정수'의 계산과 비교로 이루어진다는 점이다.

소수점이 있는 실수나 문자열보다, int가 흐름을 제어하는 뼈대가 된다. 따라서 6장의 if와 7장의 for를 제대로 다루기 위한 필수 준비물로 int가 가장 적합하다.

## 5.2 `int`란 무엇인가?

`int`는 **Integer(정수)**의 줄임말로, C언어에서 가장 기본이 되는 데이터 타입이다.

- 정수 규칙을 따른다: `int`는 소수점이 없는 딱 떨어지는 숫자만 다룬다. 계산 중에 소수점이 생겨도 **소수점 아래는 과감히 버리는** 정수만의 규칙이 적용된다.

- 크기는 유동적이다: "메모리를 몇 바이트 차지하나?"는 컴퓨터와 컴파일러마다 다를 수 있다.

  - 지금 단계에서는 **"우리가 보통 사용하는 숫자는 충분히 들어간다"** 정도로만 이해하고, **특정 크기에 의존하지 않는 습관**을 들이는 것이 더 중요하다.

## 5.3 `int` 값을 코드에 적는 방법: 정수 리터럴
코드에 값 자체를 직접 적은 것을 **리터럴(Literal)**이라고 부른다. C언어에서 정수 리터럴은 우리가 아는 숫자를 쓰는 방식과 같이 적으면 된다.

```c
0
1
10
-3
2026
```

주의할 점: 앞에 '0'을 붙이지 말자
- 지금 단계에서 가장 중요한 규칙 하나만 잡고 가자. 숫자 앞에 불필요한 0을 붙이면 안 된다.
  우리는 습관적으로 날짜나 번호를 맞출 때 01, 02, 012처럼 쓰기도 하지만, C언어는 이것을 **"전혀 다른 숫자(8진수)"**로 해석한다.

    규칙: 그냥 0은 괜찮지만, 012처럼 앞에 0이 붙은 숫자는 쓰지 않는다. (왜 그런지는 나중에 다룬다. 지금은 **엉뚱한 값이 된다**는 것만 기억하자.)

## 5.4 이름 붙인 `int` 상자: 변수
이전에 "데이터를 상자에 담고 이름을 붙인다"고 했다. C언어에서 그 "이름 붙은 상자"가 바로 **변수(variable)** 다.

### 5.4.1 변수 만들기: 선언(Declaration)
상자를 만드는 행위를 선언이라고 한다.

```c
int x;
```
- int (타입): "이 상자는 정수 규칙으로 다룬다" (크기와 용도 지정)
- x (이름): "이 상자의 이름표는 x다"

### 5.4.2 C89 규칙(중요): 선언은 무조건 맨 위에!
우리가 따르는 C89 표준에는 아주 중요한 규칙이 하나 있다. **"변수 선언은 반드시 { } 블록의 맨 처음에 몰아서 해야 한다."**

즉, 일을 시작하기 전에 **필요한 재료(변수)를 책상 위에 다 꺼내놓고 시작하는 방식**이다.

```c
int main()
{
    /* [선언부] 변수는 여기서 모두 선언한다 */
    int i;
    int sum;

    /* [실행부] 실제 동작(대입, 계산)은 그 뒤에 한다 */
    
    /* 중간에 갑자기 int k; 라고 선언하면 C89에서는 에러! */
    return 0;
}
```

참고: C99부터는 중간중간 변수를 선언하는 것이 가능하다. 우리는 C89를 따르므로, 무조건 `int i;`를 맨 위로 올린다.

5.4.3 처음 값 넣기: 초기화(Initialization)
변수(상자)를 만듦과 동시에 값을 채워 넣을 수도 있다. 이를 초기화라고 한다.

```c
int x = 10;
```

  ⚠️ 주의: **쓰레기 값 (Garbage Value)** 초기화를 하지 않고 `int x;`만 하면, 상자 안에 무엇이 들어있을까? 0이 들어있을 것 같지만, 실제로는 **알 수 없는 쓰레기 값**이 들어있습니다. 그래서 변수를 만들 때는 **가능하면 초기화를 하거나, 사용하기 직전에 반드시 값을 대입**해야 한다.


## 5.5 int 값을 바꾸는 가장 기본 동작: 대입
변수(상자)를 만들었으니, 이제 그 안에 값을 넣거나 바꿀 차례다.

### 5.5.1 대입(Assignment) 연산자 =
```c
int main() {
    int x;
    
    x = 10;
    x = 20;
}
```
여기서 사용된 = 기호를 대입 연산자라고 부른다. 수학 시간의 기억 때문에 헷갈리기 쉽지만, 프로그래밍에서는 의미가 완전히 다르다.

= (대입): 오른쪽에 있는 값(계산 결과)을 왼쪽 상자에 넣으라는 뜻.

== (비교): 양쪽이 서로 같다는 뜻. (나중에 배움)

💡 팁: 코드를 읽을 때 x = 10을 "x는 10이다"라고 읽으면 수학과 헷갈린다. "x에 10을 넣는다" 또는 **"x가 10이 된다"**라고 동작 중심으로 읽는 습관을 들이는 것이 좋다.

※ 덮어쓰기 (Overwrite)
위 코드에서 x = 10;을 실행한 뒤 x = 20;을 실행하면 어떻게 될까? 변수 상자는 한 번에 하나의 값만 담을 수 있다. 따라서 기존에 있던 10은 사라지고, 새로운 값 20만 남게 된다.

## 5.6 산술 연산자: 딱 5개만 알면 된다

C언어에는 수많은 연산자가 있지만, 계산을 위해 당장 필요한 건 **산술 연산자 5개**뿐이다.

| 연산자 | 뜻 | 설명 | 예시 (`a=10, b=3`) | 결과 |
| :---: | :---: | :--- | :--- | :---: |
| **`+`** | 더하기 | 값을 더한다. | `a + b` | `13` |
| **` - `** | 빼기 | 값을 뺀다. (또는 부호를 바꾼다) | `a - b` | `7` |
| **`*`** | 곱하기 | 값을 곱한다. (`x`가 아님에 주의) | `a * b` | `30` |
| **`/`** | **나누기 (몫)** | **몫만 구한다.** (소수점 버림) | `a / b` | **`3`** |
| **`%`** | **나머지** | 나눈 뒤 **나머지만 구한다.** | `a % b` | **`1`** |

### 5.6.1 정수와 정수가 만나면? (중요)

`int`끼리 계산하면 결과도 무조건 `int`가 나옵니다.
이 규칙 때문에 **나누기(`/`) 연산**에서 많은 오해가 생긴다.

* **수학:** $10 \div 3 = 3.3333...$
* **C언어 (`int`):** `10 / 3` = **`3`**

컴퓨터가 고장 난 게 아니다. **"정수 상자(`int`)에는 소수점이 들어갈 수 없다"**는 5.2절의 규칙 때문에 **소수점 이하가 '칼같이' 잘려 나간 것(Truncation)** 이다.

> **핵심:** `int` 세계에서 `/`는 "나누기"라기보다는 **"몫 구하기"**라고 기억하는 편이 더 정확하다.
